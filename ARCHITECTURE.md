# Architecture

High-level map of how Kommune Quiz is put together. Aimed at humans and AI agents who need to find or change something quickly.

## Tech

- **React 19** with the React compiler doing auto-memoization. Avoid hand-rolled `useMemo` / `useCallback` unless profiling shows a need.
- **TypeScript** strict mode. Types live in `src/types/`.
- **Vite 7** for dev and build.
- **d3-geo** + **topojson-client** for projection, paths, area, mesh borders.
- Plain CSS with custom properties — split into thematic files under `src/styles/` and combined by `index.css`.
- **Cloudflare Pages** deploy via `wrangler.json` (SPA fallback).

## Top-level data flow

```
App.tsx
├─ useMapData()                 → KommuneFeature[]            (TopoJSON → GeoJSON)
├─ activeFeatures               = features (optionally fylke-filtered)
├─ useDailyQuiz(features)       → today's 5-question state
├─ useMapGame / useShieldGame /
   useReverseGame(activeFeatures) — only one is "real", others get []
├─ activeQuiz                   = pick by gameMode
└─ render: DailyGame  | MapGame | ShieldGame | ReverseGame
```

The map container is a flex-column with the command bar in flow at the top and the game (which is usually a map) filling the rest. Hint rows live inside the command bar, so when they grow, the map naturally shrinks instead of being covered.

## Modes

Each mode is one folder under `src/modes/`:

| Folder | Hook | Component | Validation |
|--------|------|-----------|------------|
| `map/` | `useMapGame` | `MapGame` | Click → `kommunenummer` match |
| `shield/` | `useShieldGame` | `ShieldGame` | Typed name → kommunenummer |
| `reverse/` | `useReverseGame` | `ReverseGame` | Typed name → kommunenummer, exposes `highlightedKommune` |
| `daily/` | `useDailyQuiz` (in `hooks/`) | `DailyGame` + `DailyCommandBar` + `DailyCompletionOverlay` | Combines all three above for a 5-question run |

The mode hooks all wrap **`useQuizState`** (in `hooks/`), the shared state machine: shuffled order, current target, errors, skip, restart. Adding a new mode = a new folder + entries in `src/config/gameModes.ts` + a render branch in `App.tsx`.

## Daily quiz

`useDailyQuiz` is the orchestrator:

- Day key from `getTodayDateKey()` (Europe/Oslo).
- Question set generated by `utils/dailyChallenge.ts` using a seeded RNG. Same day → same questions for everyone.
- State persisted in `localStorage` (`utils/dailyStorage.ts`): current index, per-question errors, results, completion flag.
- Round 0 is the canonical daily; bonus rounds (`playOneMore`) are ephemeral.
- Progressive hints (`utils/dailyHints.ts` + the per-mode logic) are exposed via `DailyHints` and rendered by `components/ui/HintBar.tsx`.

## Map rendering

- **Projection**: custom Mercator in `utils/geo.ts`. Always derived from *all* features so the viewBox stays stable when filtering by fylke.
- **Paths**: `hooks/useMapPaths.ts` turns features into `{ d, kommunenummer }` rows and an active-set.
- **Zoom/pan**: `hooks/useMapZoom.ts` manipulates the SVG `viewBox` (so click coordinates stay accurate). Wheel zoom uses exponential `exp(-deltaY * f)`; trackpad pinches (detected via `ctrlKey`) use a larger factor. Touch handling does tap-vs-drag detection, pinch zoom, and double-tap-to-zoom.
- **Click delegation**: a single click handler on the map group reads `data-id` to identify the clicked kommune. Avoids per-shape event handlers.

## Hints

Computed per mode in the respective hook and surfaced by `components/ui/HintBar.tsx`:

- **Map**: each wrong guess adds a row showing the guessed kommune, direction arrow, km distance, and proximity %.
- **Shield (våpen)**: after the first error, show the kommune's area (km²) plus hangman-style letter blanks that fill in over time.
- **Reverse (omvendt)**: after the first error, show a **prefix** of the name that grows by one letter per error. Never reveals total length.

Letter reveal logic is in `utils/dailyHints.ts`. Pass `{ sequential: true }` for prefix-style (reverse), omit it for the seeded-random reveal (shield).

## Geometry math

`src/utils/geoDistance.ts`:

- `computeCentroid`, `haversineDistance`, `bearing`, `bearingToArrow` — distance hints.
- `computeAreaKm2(feature)` — area in km² via `d3.geoArea × R²`. Handles reversed winding by flipping anything above 2π steradians.

## Styles

CSS is intentionally plain. Files under `src/styles/`:

| File | Scope |
|------|-------|
| `base.css` | Variables, reset, app shell, aurora background |
| `command-bar.css` | Command bar (incl. mobile compact strip) |
| `map.css` | Map container, kommune shapes, zoom buttons |
| `modes.css` | Name input, shield game, reverse overlay, feedback |
| `daily.css` | Daily-quiz-specific UI (badge, hints, letter blanks, completion) |
| `completion.css` | Generic completion overlay |
| `index.css` | Imports the others |

Dark/light is driven by `data-theme` on `<html>` from `hooks/useTheme.ts`.

## Conventions

- **Norwegian UI text**, **English code**. Keep domain terms (`kommune`, `fylke`, `kommunenummer`, `fylkesnummer`) as-is — they appear in data and URLs.
- **Components are presentational**; logic belongs in hooks.
- **Small files** (<150 lines), single-responsibility — easier to read, easier for AI agents to rewrite.
- **Types-first**: add to `src/types/` before wiring through hooks.
- **Don't** hand-roll memoization unless you've measured a problem; the React compiler covers most cases.
- **Don't** import from `node_modules` paths or relative-up across three or more `..`.

## Data

- `data/kommuner.json` — 357 kommuner as TopoJSON. Properties: `{ kommunenummer, navn, fylkesnummer, fylkenavn }`. First two digits of `kommunenummer` equal `fylkesnummer`.
- `data/fylker.json` — 15 fylker.
- `public/shields/{kommunenummer}.png` — coat of arms. Missing shields fall back gracefully in `KommuneShield`.

Regenerate with `scripts/prepare-data.mjs` and `scripts/download-shields.mjs`.
